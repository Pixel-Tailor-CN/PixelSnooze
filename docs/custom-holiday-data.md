# 自建节假日数据指南

Pixel Snooze 支持使用用户指定的云端地址获取节假日数据。这个文件只用于回答一个问题：今天是否命中节假日休息日期。

固定每周休息规律、大小周、周期休息和完全自定义日期，请继续在应用内“休息日规则”中配置。

## 适用场景

- 使用港澳地区、美国或其他地区的公众假期。
- 使用公司、学校或团队内部发布的休息日期。
- 希望自己维护节假日数据，不使用应用默认的中国大陆数据源。

## 文件要求

云端地址需要满足以下要求：

- 使用 HTTPS URL。
- 请求返回 HTTP `2xx` 状态码。
- 返回 UTF-8 编码的 JSON 文本。
- 内容可以直接被浏览器访问，不需要登录、Cookie 或特殊请求头。
- 文件内容必须符合 Pixel Snooze 的节假日 JSON 格式。

推荐托管位置：

- GitHub 仓库中的 `holiday.json`，通过 GitHub Raw 地址访问。
- GitHub Gist Raw 地址。
- Cloudflare Pages、Netlify、Vercel 或其他静态文件服务。
- 自己维护的 HTTPS 静态文件服务器。

## 数据格式

顶层必须是数组。数组里的每一项表示一个年份：

```json
[
  {
    "year": 2026,
    "holidays": [
      "2026-01-01",
      "2026-02-16",
      "2026-07-04"
    ]
  },
  {
    "year": 2027,
    "holidays": []
  }
]
```

字段含义：

- `year`：年份，必须是整数，例如 `2026`。
- `holidays`：该年份内应被视为节假日休息日的日期数组。

日期格式要求：

- 日期必须是 `yyyy-MM-dd`。
- 示例：`2026-01-01`、`2026-07-04`。
- 年、月、日必须补齐为固定长度。
- 不要使用 `2026/01/01`、`01-01-2026`、`2026-1-1` 等格式。

## 判断规则

Pixel Snooze 运行时只判断今天是否休息：

- 今天存在于任意年份项的 `holidays` 中时，视为命中节假日休息日期。
- 今天不在 `holidays` 中时，节假日数据本身不会把今天标记为休息。
- 如果当前休息日规则是“节假日 + 双休”“节假日 + 单休”“节假日 + 大小周”或“节假日 + 周期”，应用会继续叠加对应的本地规则。
- 如果当前休息日规则是“完全自定义”，应用只使用用户在本地手动标记的日期。

## 推荐维护范围

建议每次维护以下年份：

- 去年：方便跨年后回看和短期过渡。
- 今年：保证当前判断可用。
- 明年：避免新年后短期没有数据。

如果未来年份还没有发布，可以先保留该年份并使用空数组：

```json
[
  {
    "year": 2027,
    "holidays": []
  }
]
```

空数组表示该年份暂时没有由节假日数据标记的休息日期。

## 生成建议

推荐流程：

1. 从当地官方或可信来源收集实际休息日期。
2. 只保留需要 Pixel Snooze 视为休息日的日期。
3. 按年份分组写入 `year` 和 `holidays`。
4. 对每个日期使用 `yyyy-MM-dd` 格式。
5. 按日期从早到晚排序，方便人工检查。
6. 使用 JSON 校验工具确认文件合法。
7. 上传到 HTTPS 静态地址。
8. 在 Pixel Snooze 设置页填写该地址并保存。
9. 等待应用完成可用性检测。检测通过后，应用会立即缓存一份数据并保存 URL。
10. 回到“调休日历”确认年份范围和休息日数量符合预期。

## GitHub 托管示例

假设仓库路径为：

```text
https://github.com/example/holiday-data/blob/main/holiday.json
```

应用中应填写 Raw 地址：

```text
https://raw.githubusercontent.com/example/holiday-data/refs/heads/main/holiday.json
```

首次保存该 URL 时，Pixel Snooze 会先检测地址并缓存一份可用数据。之后如果你更新了云端文件，Pixel Snooze 不会自动后台拉取，需要在应用内手动点击“调休日历”的“更新”按钮。

## 检查清单

保存 URL 前，建议确认：

- 浏览器可以直接打开这个 URL。
- 打开后看到的是 JSON 文本，不是网页预览。
- JSON 顶层是数组。
- 每个年份项都有 `year` 和 `holidays`。
- 每个日期都使用 `yyyy-MM-dd`。
- 文件中包含当前年份。
- 当前日期如果应该休息，已经出现在对应年份的 `holidays` 中。

## 常见问题

### 可以在 JSON 里写节日名称吗？

不需要。Pixel Snooze 当前只读取 `year` 和 `holidays`，不会显示或使用节日名称。

### 可以只维护一个年份吗？

可以。只要当前日期所在年份的数据存在，并且需要休息的日期已经写入 `holidays`，应用就能判断今天是否命中节假日休息日期。

### URL 改了以后会立即生效吗？

会先检测再生效。保存 URL 时，Pixel Snooze 会请求该地址、校验 JSON 格式并写入本地缓存。只有检测和缓存成功后，URL 才会保存。

### 检测或更新失败会怎样？

保存自定义 URL 时检测失败，Pixel Snooze 不会保存该 URL，也不会替换当前可用数据。后续手动更新失败时，Pixel Snooze 会保留当前可用的本地缓存或内置数据。请检查 URL 是否能直接访问、HTTP 状态码是否正常、JSON 格式是否正确。

### 恢复默认会怎样？

恢复默认会清除当前节假日数据缓存并回到应用内置数据源。之后如果你在“调休日历”里手动更新，应用会重新请求默认云端数据地址。
