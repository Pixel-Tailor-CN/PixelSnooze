# 临时休息模式设计

## 背景

用户可能为工作日设置多个闹钟。临时请假或度假时，逐个关闭并在假期结束后重新开启容易遗漏；临时修改既有休息日规则也需要多步操作。

临时休息模式用于在不修改系统闹钟、不覆盖既有休息日规则的前提下，临时把今天视为休息日。关闭或到期后，Pixel Snooze 继续使用用户原有的休息日规则。

## 目标

- 在应用首页提供清晰、可恢复的临时休息入口。
- 提供快捷设置磁贴，实现常用场景的一次点击操作。
- 模式开启时显示状态通知，降低用户忘记关闭的风险。
- 提供应用图标长按快捷方式。
- 所有入口共享同一份本地状态，行为保持一致。
- 通知监听热路径只读取轻量本地状态，不加入网络 I/O 或重型解析。

## 非目标

- 不创建、关闭、修改或删除系统闹钟。
- 不修改、备份或恢复用户既有休息日规则。
- 不增加桌面小组件。
- 不向 Tasker 或其他应用开放广播、Service 或其他无交互自动化接口。
- 不引入数据库、云端同步、前台服务或精确闹钟权限。
- 不判断明天或其他日期是否应该跳过闹钟。运行时仍然只判断今天。

## 状态模型

新增独立的临时休息状态：

```kotlin
sealed interface TemporaryRestState {
    data object Disabled : TemporaryRestState
    data class UntilDate(val endDate: LocalDate) : TemporaryRestState
    data object UntilDisabled : TemporaryRestState
}
```

含义如下：

- `Disabled`：未开启。
- `UntilDate`：从现在起至 `endDate` 当天结束前有效，结束日期包含当天。
- `UntilDisabled`：一直有效，直到用户手动关闭。

快捷入口的“仅今天”保存为 `UntilDate(LocalDate.now())`。

运行时只读取当前本地日期：

```text
状态为 UntilDisabled，今天按休息日处理
状态为 UntilDate 且今天不晚于结束日期，今天按休息日处理
否则临时休息未生效，继续执行既有休息日规则
```

日期状态过期后视为 `Disabled`。读取或执行状态同步时可以顺便清理过期数据，但不依赖后台定时任务保证业务正确性。

## 本地持久化

继续使用 `SharedPreferences`，新增独立的 `TemporaryRestPreferencesRepository`。建议保存：

- `temporary_rest_mode`：`DISABLED`、`UNTIL_DATE` 或 `UNTIL_DISABLED`。
- `temporary_rest_end_date`：`UNTIL_DATE` 对应的 ISO-8601 日期。

缺失、无法识别或损坏的数据回退为 `Disabled`，避免意外跳过闹钟。

## 统一状态入口

新增 `TemporaryRestController`，负责所有状态变更及变更后的同步：

- 开启仅今天。
- 开启至指定日期。
- 开启直到手动关闭。
- 关闭。
- 读取并清理过期状态。
- 请求刷新快捷设置磁贴。
- 发布或取消状态通知。

首页、磁贴、通知操作和图标快捷方式都通过该控制器变更状态，避免各入口分别实现业务规则。

`RestDayRepository.isRestDay()` 在读取既有规则前先检查临时休息状态：

```text
临时休息当前有效，返回 true
否则继续执行现有休息日判断
```

关闭或到期后不需要执行“恢复配置”，因为原配置从未被修改。

## 首页状态卡

在首页主要状态区顶部增加“临时休息模式”卡片。

关闭状态：

```text
临时休息模式                         [关闭]
临时请假时，将今天直接按休息日处理
```

开启状态：

```text
临时休息模式                         [开启]
仅今天有效
```

或：

```text
临时休息模式                         [开启]
有效至 7 月 26 日（含当天）
```

交互规则：

- 直接打开开关时默认开启“仅今天”，保持一次点击即可完成高频操作。
- 关闭开关立即关闭临时休息，不弹确认框。
- 点击卡片正文打开持续时间对话框。
- 对话框提供“仅今天”“选择结束日期”“直到手动关闭”和“关闭”。
- 选择的结束日期不得早于今天。
- 页面从后台恢复时重新读取状态，确保磁贴、通知或快捷方式造成的变更及时显示。
- 卡片提供“添加快捷设置磁贴”入口。Android 13 及以上调用系统添加磁贴请求；Android 12 显示手动添加说明。

如果 Android 13 及以上尚未授予通知权限，用户首次从应用内开启临时休息模式时请求通知权限。拒绝权限不影响临时休息模式本身。

## 快捷设置磁贴

新增可切换的 `TileService`，标签为“临时休息”。

状态展示：

- 未生效：`STATE_INACTIVE`，副标题“已关闭”。
- 当前有效：`STATE_ACTIVE`，副标题显示“仅今天”“至 7 月 26 日”或“手动关闭”。

点击规则：

- 未开启时单击：开启“仅今天”。
- 已开启时单击：立即关闭。
- 长按：通过 `ACTION_QS_TILE_PREFERENCES` 打开应用内持续时间设置。

安全规则：

- 设备安全锁定时，开启临时休息模式前要求用户解锁。
- 关闭临时休息模式不会增加漏闹钟风险，可以直接执行。
- 磁贴每次开始监听时重新读取状态，避免展示过期状态。

Android 13 及以上允许用户在首页通过系统弹窗添加磁贴；更低版本由用户在快捷设置编辑页手动添加。

## 状态通知

仅在临时休息模式当前有效时显示静默、常驻状态通知：

```text
临时休息模式已开启
今天的目标闹钟将按休息日处理

[关闭] [修改时长]
```

通知内容根据状态显示“仅今天有效”“有效至某日”或“直到手动关闭”。

行为规则：

- “关闭”通过应用内不可导出的接收器调用统一控制器，并取消通知。
- “修改时长”打开首页的持续时间对话框。
- 通知不启动前台服务，不播放声音，不振动，并使用独立低打扰通知渠道。
- 有结束日期时为通知设置截至结束日期次日零点的超时；应用、磁贴或通知监听服务再次运行时也会校正过期通知。
- 用户未授予通知权限或主动关闭该通知渠道时，临时休息判断仍然正常工作。

## 应用图标长按快捷方式

提供三个静态快捷方式：

- “临时休息一天”：未开启时开启仅今天；如果已经开启到更晚日期或直到手动关闭，则保持较长的现有状态，不缩短有效期。
- “关闭临时休息”：立即关闭。
- “管理临时休息”：打开首页持续时间对话框。

快捷方式通过明确指向本应用 `MainActivity` 的 Intent 执行。应用会打开并展示变更后的状态，不对外暴露可由其他应用静默调用的状态变更组件。

## 状态同步

状态变更后的同步顺序：

```text
用户入口
  -> TemporaryRestController 写入本地状态
  -> 更新或取消状态通知
  -> 请求快捷设置磁贴刷新
  -> 当前可见的首页在生命周期恢复或本地回调时刷新
```

通知监听到目标闹钟时：

```text
读取临时休息状态
  -> 当前有效：今天是休息日
  -> 未生效：继续执行既有休息日规则
```

通知热路径只包含 `SharedPreferences` 和当前日期读取。

## 异常与兼容处理

- 状态数据损坏：回退为关闭。
- 结束日期已经过去：按关闭处理并在方便时清理。
- 通知权限被拒绝：不显示状态通知，其他入口正常工作。
- 通知渠道被关闭：不反复请求权限或阻止功能。
- 添加磁贴请求失败或被拒绝：保留手动添加说明，不重复打扰用户。
- 不同厂商对磁贴副标题或颜色的展示可能不同，主要状态仍由 `STATE_ACTIVE` 和 `STATE_INACTIVE` 表达。
- 进程重启、设备重启后从本地状态恢复，不依赖常驻进程。

## 验证

项目不新增单元测试。完成后执行：

```powershell
.\gradlew.bat :app:assembleDebug
.\gradlew.bat :app:lintDebug
```

人工检查：

- 首页开关能够开启仅今天，并能立即关闭。
- 能够选择结束日期或直到手动关闭。
- 开启、关闭和过期均不修改原有休息日规则。
- 临时休息有效时，通知监听只把今天视为休息日。
- 首页、磁贴、状态通知和图标快捷方式显示同一状态。
- 锁屏状态不能在未解锁时开启磁贴。
- 通知权限被拒绝时核心功能仍然有效。
- “临时休息一天”不会缩短已有的多日或手动关闭状态。
- 模式到期后恢复既有规则，日期通知能够消失或在下次运行时被校正。
- Android 12 和 Android 13 及以上的磁贴添加流程符合各自能力。
