# AGENTS.md

## 适用范围

本文件适用于整个仓库。所有 AI 代理在修改、审阅或生成本项目内容时，都应优先遵守本文件要求。

## 交流语言

- 与用户沟通时使用中文。
- 生成或修改代码注释时使用中文。
- 生成或修改项目文档时使用中文。
- Logcat、运行日志和调试日志内容使用英文。

## 项目核心语义

- Pixel Snooze 运行时只判断今天是否是休息日。
- 不要把业务逻辑改成判断明天或其他日期。
- 节假日数据只需要 `holidays` 休息日集合。
- 不需要 `workdays`、调休补班日或工作日数据。
- 日期不在 `holidays` 中时，视为非休息日。

## 开发约束

- 保持当前本地优先、低侵入实现，不引入数据库、云端同步、Root、Xposed 或反编译能力，除非用户明确要求。
- 通知监听热路径中不要加入网络 I/O 或重型解析逻辑。
- 不要新增单元测试或要求补充单元测试，除非用户明确要求。
- 不要为了测试覆盖率改动项目结构。
- 优先保持改动小而直接，避免无关重构。

## 验证方式

本项目默认验证方式为 Android 构建、lint 和必要的人工检查。

常用命令：

```powershell
.\gradlew.bat :app:assembleDebug
.\gradlew.bat :app:lintDebug
```

需要验证 Release 包时运行：

```powershell
.\gradlew.bat :app:assembleRelease
```

## 文档一致性

涉及功能说明、设计文档或 README 时，应保持以下描述一致：

- 判断对象是今天是否休息。
- 节假日 JSON 只包含 `year` 和 `holidays`。
- 不描述 `workdays`、调休补班日或默认周末规则。
- 不把单元测试列为项目要求。
