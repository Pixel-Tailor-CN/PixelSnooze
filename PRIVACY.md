# Pixel Snooze 隐私政策

更新日期：2026 年 6 月 15 日

本隐私政策适用于 Pixel Snooze（以下简称“本应用”）。本应用由 Pixel Tailor CN 维护，是一款用于在休息日辅助跳过闹钟通知的 Android 工具。

本应用以本地处理为主，不提供账号系统，不接入广告、统计分析、第三方登录、支付、定位、通讯录、相机、麦克风等功能。

## 一、我们访问或处理的信息

### 1. 通知信息

本应用需要用户在系统设置中手动授予“通知监听”权限后才能工作。授予该权限后，Android 系统允许本应用接收设备上的通知事件。

本应用的实际处理逻辑仅针对以下目标时钟应用包名：

- `com.google.android.deskclock`
- `com.android.deskclock`

对于非目标应用的通知，本应用会直接忽略，不读取和保存其通知标题、正文或操作内容。

对于目标时钟应用通知，本应用可能会读取以下信息，用于判断是否需要跳过闹钟：

- 通知所属应用包名
- 通知标题
- 通知正文
- 通知中的操作按钮标题和对应操作

这些信息仅用于本地判断通知是否匹配用户设置的关键词、今天是否为休息日，以及是否存在可用的“跳过/关闭”操作。

### 2. 用户设置

本应用会在设备本地保存用户配置，包括：

- 闹钟通知关键词
- 跳过按钮文本匹配规则
- 休息日规则配置，例如单休星期几、大小周配置、上 x 休 y 周期锚点
- 完全自定义模式下用户手动标记的上班和休息日期

这些配置仅保存在本应用的本地私有存储中，不会由本应用主动上传到服务器。

### 3. 执行记录

为了让用户了解本应用最近的执行情况，本应用会在设备本地保存最近若干条记录，包括：

- 自动跳过闹钟记录
- 检测到目标闹钟通知后的处理记录
- 通知所属应用包名
- 通知标题和正文
- 执行时间
- 执行动作，例如点击跳过或忽略通知
- 执行原因，例如关键词未匹配、今天未命中休息日规则、未找到跳过动作等

这些记录仅用于在本应用主页面展示，不会由本应用主动上传到服务器。

### 4. 调休日历数据

本应用内置一份调休日历数据，并支持用户手动从云端更新调休日历。调休日历数据只包含年份和休息日日期，不包含个人信息。

用户选择的休息日规则和完全自定义日期不属于调休日历数据，它们只保存在本地私有存储中，不会由本应用主动上传。

当用户手动触发更新时，本应用会请求以下地址获取最新调休日历数据：

https://raw.githubusercontent.com/Pixel-Tailor-CN/PixelSnooze/refs/heads/main/json/holiday.json

更新后的调休日历数据会保存在本应用的本地私有存储中。应用运行时会优先使用本地已更新的数据；如果本地没有更新数据，则使用应用内置数据。

本应用不会在启动时自动后台拉取调休日历数据。

## 二、我们如何使用这些信息

本应用访问或处理上述信息，仅用于以下目的：

- 判断目标时钟通知是否包含用户设置的关键词
- 判断今天是否为休息日
- 查找通知中可用于跳过或关闭闹钟的操作按钮
- 在满足条件时触发通知中的跳过或关闭操作
- 在主页面展示最近的自动跳过记录和通知处理记录
- 手动更新并缓存调休日历数据

本应用不会将这些信息用于广告、用户画像、行为追踪、营销推荐或其他与核心功能无关的目的。

## 三、数据共享与第三方服务

本应用不会主动向开发者服务器上传用户的通知内容、用户设置、执行记录或其他个人数据。

本应用可能涉及以下第三方网络访问：

- 用户手动更新调休日历时，请求 GitHub Raw 上的 `holiday.json` 文件。
- 用户点击应用内的 GitHub、Pixel Tailor 或 Telegram 等外部链接时，系统会使用浏览器或其他应用打开对应网页。

访问第三方网站或服务时，对方可能会根据其自身隐私政策处理网络请求信息，例如 IP 地址、浏览器信息或访问日志。此类处理不由本应用控制。

## 四、数据保存与删除

本应用保存的数据位于设备本地，包括用户设置、休息日规则、执行记录和手动更新后的调休日历缓存。

用户可以通过以下方式删除这些本地数据：

- 在 Android 系统设置中清除本应用数据。
- 卸载本应用。

清除应用数据或卸载应用后，本应用保存在设备本地的用户设置、休息日规则、执行记录和调休日历缓存会被删除。

## 五、权限说明

本应用使用以下权限或系统能力：

### 通知监听权限

用途：监听目标时钟应用通知，并在符合条件时尝试触发通知中的跳过或关闭操作。

授权方式：用户需要在 Android 系统设置中手动开启。用户也可以随时在系统设置中关闭该权限。

### 网络访问权限

用途：在用户手动触发调休日历更新时，从 GitHub Raw 下载调休日历数据；以及打开用户主动点击的外部链接。

本应用不会在启动时自动后台更新调休日历。

## 六、数据安全

本应用将用户设置、休息日规则、执行记录和调休日历缓存保存在 Android 应用私有存储中。通常情况下，其他普通应用无法直接访问这些数据。

本应用不会主动上传通知内容和执行记录。但在系统调试环境中，本应用可能会向 Android 系统日志写入用于故障排查的运行日志。系统日志的访问范围取决于设备系统、调试状态和开发者工具环境。

## 七、儿童隐私

本应用不是面向儿童设计的应用，也不会主动收集儿童个人信息。

## 八、政策变更

如果本应用的功能、权限或数据处理方式发生变化，本隐私政策也可能相应更新。更新后的隐私政策会通过项目仓库或应用发布渠道提供。
